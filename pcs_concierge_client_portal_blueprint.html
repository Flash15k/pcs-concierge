WHY THIS STRUCTURE WORKS

• Low cognitive load

The spouse sees only six clear navigation choices. Fewer choices create calm and make the portal feel managed rather than overwhelming.

• Emotionally reassuring

Every page answers one anxiety: what happens next, where my documents live, what I still need to do, and who can help me right now.

• Operationally scalable

This architecture is simple enough for a VA to maintain yet premium enough to support higher pricing and national expansion.

Build notes for your web designer

GLOBAL UI RULES

1. Use the approved Image 2 logo system.

2. Desktop layout: left navigation + wide main content. 3. Mobile layout: stacked cards with generous spacing. 4. Keep typography elegant in headers, simple in body copy. 5. White / cream content areas with navy and gold accents Key reusable components

only.

6. Every page should answer one client anxiety and one next

EXPERIENCE RULES

The spouse should feel the portal is calm, private, and already organized for her. Avoid clutter, dense copy, or tech-heavy wording. The interface should feel closer to a luxury concierge dashboard than a generic support portal.

step.

Status strip A gold or soft-cream alert bar for the next milestone.

Summary card Small cards for move date, status, concierge team, and response time.

File card Uniform cards with file type, title, and open/download actions.

Update card Chronological communication blocks with timestamp and clean writing.

Task row Single responsibility rows with due date, priority, and one CTA.

Message module Secure contact block with concise subject and message fields.

Deliverable note

This PDF is a visual blueprint. Your web designer can replicate it in Softr, Webflow, HighLevel, Figma, or a custom build. The structure is intentionally simple so it can be implemented quickly without losing the premium brand feel.
